# AI 줄넘기 자율학습 서비스 — 프로젝트 명세서

> Claude Code 개발 컨텍스트용 문서  
> 버전: v0.1 | 최초 작성: 2025

---

## 1. 프로젝트 개요

### 목적
중고등학교 체육 수업의 **단체 줄넘기 / 음악줄넘기** 활동에서 AI로 학생 개개인의 자세를 실시간 분석·피드백하고, 음악 박자 정확도를 측정하여 **교사의 수업 퀄리티 향상**을 지원하는 교육용 AI 시스템.

### 핵심 사용자
- **학생**: 중고등학생, 10~20명 단위 수업 참여, 실시간 피드백 수신
- **교사**: 체육 교사, 학생 개별 현황 모니터링 및 대시보드 조작

---

## 2. 개발 환경

### 하드웨어 (프로토타입)
- **현재 단계**: 노트북 내장 카메라 (단일 인원 테스트)
- **목표 단계**: 거치형 전용 카메라 + 기존 모니터 활용
- **카메라 후보**:
  - Raspberry Pi AI Camera (IMX500, $70) — 온디바이스 포즈 추정, Pi CPU 부담 없음 ★추천
  - Raspberry Pi Camera Module 3 Wide (IMX708, $35) — 120도 광각, 1080p/50fps
  - ELP USB 카메라 (OV4689) — 1080p/60fps, USB 범용
  - 저가 USB 광각 카메라 — 120도, 1080p/30fps, 예산 절감용
- **메인 컴퓨터 후보**: Raspberry Pi 5 (8GB, ~14만원) 또는 Intel N100 미니 PC
- **예산**: 프로토타입 30만원 이내 (모니터 제외)
- **마이크**: USB 마이크 (음악 박자 분석용)

### 소프트웨어 스택
```
Language  : Python 3.10+
Vision    : MediaPipe (포즈 추정), OpenCV (영상 처리)
Audio     : librosa 또는 aubio (BPM/박자 감지), PyAudio (마이크 입력)

Dashboard : Tkinter 또는 Flask + 웹UI (교사 대시보드)
AI API    : Anthropic Claude API (자세 분류 고도화, 선택적)
Storage   : SQLite (로컬 수업 기록)
```

## 3. 기능 요구사항

### 3.1 포즈 추정 및 자세 분석

```
입력  : 카메라 영상 프레임 (노트북 웹캠 → 전용 카메라)
처리  : MediaPipe Pose로 신체 33개 키포인트 추출
출력  : 관절 좌표, 자세 점수(0~100), 오류 유형
```

**분석 항목**
- 팔꿈치 각도 및 팔 회전 궤적
- 점프 높이 및 발 착지 방식 (발끝 vs 발뒤꿈치)
- 상체 기울기 및 중심 축 안정성
- 줄 통과 타이밍과 신체 동작 동기화

**다중 인원 처리** (Phase 2)
- 10~20명 동시 감지 및 개인별 추적
- 화면 내 위치 기반 학생 ID 매핑

### 3.2 횟수 자동 카운팅

```
방법  : 발목/무릎 키포인트 Y좌표 변화로 점프 사이클 감지
종목  : 단순뛰기, 이단뛰기, 교차뛰기
목표  : 카운팅 정확도 95% 이상
```

**알고리즘 개요**
1. 발목 Y좌표의 상승/하강 사이클 감지
2. 임계값(threshold) 기반 점프 판정
3. 연속 프레임 필터링으로 오탐 방지
4. 종목별 팔 동작 패턴으로 이단/교차 구분

### 3.3 음악줄넘기 박자 분석

```
입력  : 마이크로폰 실시간 음성 (수업 현장 음악 수음)
처리  : BPM 감지 → 박자 그리드 생성 → 점프 타이밍 동기화
출력  : 박자 정확도 점수, 이탈 패턴 (빠름/늦음/불규칙)
목표  : 박자 감지 오차 ±50ms 이내
```

### 3.4 실시간 피드백 코칭

**화면 피드백**
- 학생별 포즈 키포인트 오버레이
- 자세 점수 및 박자 정확도 게이지
- 오류 부위 관절 하이라이트 (색상 강조)

### 3.5 교사 대시보드

- 전체 학생 자세 점수 실시간 현황 (리스트/히트맵)
- 박자 정확도 분포 시각화
- 횟수 집계 실시간 표시
- 수업 기록 저장 및 조회 (SQLite)
- [TBD] 수행평가 데이터 내보내기 (CSV)

## 7. 주요 설정값 (config.py 예시)

```python
# 카메라
CAMERA_INDEX = 0          # 0: 노트북 내장 카메라
FRAME_WIDTH = 1280
FRAME_HEIGHT = 720
TARGET_FPS = 30

# 포즈 추정
POSE_CONFIDENCE = 0.7     # MediaPipe 최소 신뢰도

# 횟수 카운팅
JUMP_THRESHOLD = 0.05     # 발목 Y좌표 변화 임계값 (정규화)
MIN_JUMP_FRAMES = 3       # 최소 점프 인정 프레임 수
COOLDOWN_FRAMES = 10      # 연속 카운팅 방지 쿨다운

# 박자 분석
AUDIO_SAMPLE_RATE = 44100
BPM_HOP_LENGTH = 512
BEAT_TOLERANCE_MS = 50    # 박자 오차 허용 범위
```
